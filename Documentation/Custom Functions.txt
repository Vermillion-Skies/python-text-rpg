----------------------------
CUSTOM FUNCTIONS DOCUMENTATION
ONLY functions included in Scripts.py
----------------------------
----------------------------
screenclear()
Runs a short loop of empty lines to clear a terminal screen for more content. When called, it sets the "rep" variable to integer 0. While the "rep" variable is less than 100, it will print an empty line and increment this variable by 1. Once the loop is done, it returns to the process that called it.
----------------------------
----------------------------
savegame(x)
Function used to save the contents of a provided list (x, which henceforth will be simply referred to as "list" or "the list") to a .txt file (save.txt, which henceforth will be simply referred to as "save" or "the file"). When called, it sets the "activeinput" variable to 1, as to serve as a placeholder for the loop while it processes data. It first checks if the "Saves/" folder exists. If this folder doesn't exist, it makes it. Then it will attempt to write the list to the save, with a dedicated handler if this fails. Upon completion or upon error, it will cease and return to the process that called it.
----------------------------
----------------------------
loadgame()
Function used to load variables from a .txt file (save.txt, which henceforth will simply be referred to as "save" or "the file") and return it as a list. When called, it verifies the existence of the "Saves/" folder, throwing an error if it cannot be called. It then continues to verify if the save file exists, throwing an error if that fails. Once both of these are verified, it attempts to write the contents of the save file to a local list, throwing an error if that causes an exception. If all checks pass and the program can write to the list, it returns the list to the process that called it, before terminating itself.
----------------------------
----------------------------
errorhandle(x)
Function used to catch and explain errors thrown by the program and any of its subprocesses. When it's called, the caller declares an error code (x) and begins the error handler. The handler will halt the program, clear the terminal, and act as a stop point so the program doesn't overwhelm a user's system. It then displays a short message alongside the error code, calling upon the user to check the documentation and confirm a program exit.
----------------------------
----------------------------
statcheck(x, y)
Function used to check two stats against each other, with x being the indicated minimum value, and y being the provided value. Should x be greater than y, it returns a fail string. Should y be greater than or equal to x, it returns a pass string.
----------------------------
----------------------------
checkcache(x)
Function used to load a variable from the local cache. It loads the entire cache file into a list variable, then checks and returns the requested value at location x.
----------------------------
----------------------------
writecache(x, y)
Function used to write values to the cache. Loads the entire cache into a local list variable, then writes value y to location x. After this happens and passes, it saves the list once more to a file.
----------------------------
----------------------------
sprint(x)
Function used to display text in the terminal one character at a time, as opposed to all at once. When called, for the length of input x, it will print each character (c), run flush to output immediately, and sleep for 0.05 seconds each letter. Once complete, it'll sleep for another half second before printing a newline character and returning to the function that called it.
----------------------------
----------------------------
gameover()
Function to have a unified game over dialog throughout the game. Calls sprint and prints a few lines of text before calling an unbreakable loop forcing restart.
----------------------------
----------------------------
loadchapter(x)
Function to load a chapter defined by the caller. A simple nested conditional to check input and then load a chapter based on said input
